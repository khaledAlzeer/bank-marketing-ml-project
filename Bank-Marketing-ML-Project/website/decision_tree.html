📌 What is a Decision Tree?

Understanding the algorithm

A Decision Tree is a flowchart-like model that splits data into branches based on feature values. Each internal node represents a decision rule, each branch represents an outcome, and each leaf node represents a class label.

The tree is built by selecting the feature that provides the best split at each node using criteria such as Gini Impurity or Information Gain (Entropy).

Decision Trees are highly interpretable — you can literally follow the path from root to leaf to understand why a prediction was made. However, they can easily overfit without proper pruning.

Gini Impurity: Gini = 1 − Σ(pᵢ)²

✅ Advantages

  • Highly interpretable
  • No feature scaling needed
  • Handles non-linear patterns
  • Works with mixed data types

❌ Limitations

  • Prone to overfitting
  • Unstable — small data changes alter tree
  • Biased toward features with more levels

💻 Training the Model

How we trained the Decision Tree

01

Load Data

# Load preprocessed data
train = pd.read_csv('train.csv')
test = pd.read_csv('test.csv')

print("✅ Data loaded successfully!")
print("Train Shape:", train.shape)
print("Test Shape:", test.shape)
02

Split Data

# Split Features and Target
X_train = train.drop('y', axis=1)
y_train = train['y']

X_test = test.drop('y', axis=1)
y_test = test['y']

print("X_train:", X_train.shape)
print("y_train:", y_train.shape)
print("X_test:", X_test.shape)
print("y_test:", y_test.shape)
03

Train Model

from sklearn.tree import DecisionTreeClassifier

# Train Model
dt_model = DecisionTreeClassifier(
    random_state=42
)
dt_model.fit(X_train, y_train)

print("✅ Decision Tree trained successfully!")

What do the numbers tell us?

What Worked Well

Key Takeaway

  • Single tree is a weak learner
  • Performs worse than Logistic Regression
  • Ensemble methods fix instability

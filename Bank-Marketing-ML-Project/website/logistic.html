📌 What is Logistic Regression?

Understanding the algorithm

Logistic Regression is a statistical model used for binary classification tasks. Despite its name, it is a classification algorithm — not regression.

It uses the sigmoid function to map any input value to a probability between 0 and 1, then applies a threshold (usually 0.5) to classify the output as 0 or 1.

In this project, Logistic Regression serves as our baseline model — a reference point to compare more complex models against.

Sigmoid Function: P(y=1) = 1 / (1 + e-z)

✅ Advantages

  • Fast to train
  • Highly interpretable
  • Works well on linearly separable data
  • Outputs calibrated probabilities

❌ Limitations

  • Assumes linear decision boundary
  • Struggles with complex patterns
  • Sensitive to outliers

How we trained Logistic Regression

Load Data

02

Split Features & Target

# Split Features & Target

X_test = test.drop('y', axis=1)
y_test = test['y']

print("X_train:", X_train.shape)
print("X_test :", X_test.shape)
03

Train Model

from sklearn.linear_model import LogisticRegression

# Train Model

    max_iter=1000,
    random_state=42

print("✅ Model trained successfully!")

🔎 Result Analysis

What do the numbers tell us?

✅

What Worked Well

  • Strong overall accuracy of 87.6%
  • Excellent performance on Class 0 (F1: 0.93)
  • High recall for negatives: 0.94
  • Fast training — converged quickly
  • AUC of 0.73 — decent discrimination
⚠️

What Could Be Better

  • Low recall for Class 1: 0.38
  • Only 398 true positives out of 1,058 actual subscribers
  • F1-Score for "Yes" class is only 0.42
  • Linear boundary limits complex pattern detection
  • Misses many actual subscribers (660 false negatives)
💡

Key Takeaway

  • Good baseline — sets the performance bar
  • Trade-off: high accuracy but low recall on minority class
  • Linear model struggles with SMOTE-balanced data
  • More complex models needed for better recall
  • Next: try tree-based models for non-linear patterns
